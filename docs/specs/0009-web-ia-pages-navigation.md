# Спека 0009 — Web IA: страницы, навигация и «5 минут ценности» (DRAFT)

Статус: **draft**

## Цель

Сделать веб-интерфейс операционной платформой студии: за 5 минут PM/Owner должен понять состояние проекта и сделать действие.

## Ключевой UX-инвариант

Пользователь **всегда** должен понимать:

- какой проект активен
- что нового/что горит
- что нужно сделать дальше

## Навигационная модель

Минимальная структура:

- **Login**
- **Projects** (создать/выбрать активный)
- **Dashboard** (по активному проекту)
- **Conversations** (ридер переписки + evidence)
- **Commitments**
- **Risks**
- **Digest**
- **Jobs**
- **Settings** (интеграции и связывания)

## «5 минут ценности» (целевой сценарий)

1) Открыть продукт
2) Выбрать активный проект (если не выбран)
3) Увидеть:
   - краткий статус
   - топ commitments (просроченные/ближайшие)
   - топ риски
   - последнюю активность переписки
4) Сделать действие:
   - подтвердить/закрыть commitment
   - принять следующий шаг по риску
   - запустить импорт/эмбеддинги
   - сгенерировать дайджест

## Страницы (семантика)

### 1) Login

- простой вход для внутреннего инструмента
- после входа редирект в Projects или Dashboard (если уже есть активный проект)

### 2) Projects

- список проектов
- создание проекта
- выбор активного проекта

Инвариант: без активного проекта остальные страницы либо недоступны, либо показывают CTA “выберите проект”.

### 3) Dashboard (Project Overview)

Должен отвечать на вопрос: **«что происходит и что делать»**.

Блоки (по умолчанию, но потенциально настраиваемые):

- Просроченные commitments
- Commitments на ближайшие 7 дней
- Топ рисков (severity/probability)
- Активность переписки (7 дней)
- Последняя синхронизация/состояние памяти
- Быстрые действия

### 4) Conversations

- ридер переписки
- поверх ленты — “слои смысла”:
  - commitments, связанные с сообщениями
  - evidence для рисков

Инвариант: пользователь может открыть первоисточник любого вывода.

### 5) Commitments

- разделение: активные / предложенные / закрытые
- быстрые действия: подтвердить, done, cancel, edit
- фильтры: due/overdue

### 6) Risks

- активные / предложенные / закрытые
- severity/probability/impact
- next_action
- evidence

### 7) Digest

- последний дайджест
- история версий
- кнопка “сгенерировать сейчас”

### 8) Jobs

- статусы джоб
- кнопки запуска
- счётчики памяти (pending/ready/failed)
- ошибки и причины (нет привязок, лимит, ключи)

### 9) Settings

Секции:

- Связывание с источниками (Chatwoot inbox ↔ проект)
- Интеграции (Linear/Attio) — статусы и доступы
- Политики (экономный режим, лимиты)
- Приватность/retention (как минимум отображение политики)

## Пустые состояния (очень важно)

### Нет проектов

- показывать CTA “создать проект”

### Нет привязок к источникам

- объяснить: “память не наполнится, пока не свяжете источник”

### Нет данных/памяти

- CTA “запустить импорт” и “построить память”

## Acceptance criteria

1. Пользователь заходит и без инструкции понимает, что сделать, чтобы получить данные.
2. Dashboard даёт управленческую картину за 1 экран.
3. Любой вывод кликабелен до evidence.
4. Без активного проекта нет риска смешивания данных.
