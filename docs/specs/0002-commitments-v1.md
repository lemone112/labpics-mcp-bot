# Спека 0002 — Commitments v1 (DRAFT)

Статус: **draft**

## Цель

Автоматически выделять **договорённости / обязательства / следующие шаги** (commitments) по каждому проекту на основе переписки.

## Семантика

Commitment — это утверждение вида «кто-то (мы/клиент/непонятно) сделает X к сроку Y». Оно должно быть:

- привязано к проекту
- подтверждаемо доказательством (evidence-first)
- редактируемо человеком

## Поведение

- Система периодически (или по кнопке) анализирует новую память проекта и извлекает commitments.
- Commitments появляются в списке проекта автоматически.
- Повторный прогон не должен создавать дубликаты.

## UX

- У проекта есть список commitments.
- Можно:
  - отметить выполненным
  - отменить
  - поправить формулировку/срок/ответственного

## Acceptance criteria

- Каждый commitment содержит ссылку на первоисточник.
- Список не захламляется дублями.
- Commitments строго изолированы по проектам.
