# Спека 0005 — Интеграции Linear/Attio: preview/apply (DRAFT)

Статус: **draft**

## Цель

Дать интеграции, которые помогают действовать, но не ломают данные.

## Принцип

- **Attio (CRM)**: только preview → approve → apply.
- **Linear (PM)**: в v1 всё через preview/approve (авто — позже и только в узком безопасном случае).

## Семантика «предложения действия»

Предложение действия — это объект, который содержит:

- что именно изменится (diff смыслом)
- почему это предлагается (объяснение)
- на основании чего (evidence)
- риск/уверенность
- что будет, если не применить

## Поведение

- Система формирует предложения действий (создать задачу, обновить поле, добавить комментарий).
- Пользователь видит предложения и подтверждает.
- Система применяет действие и фиксирует результат.

## Аудит (обязателен)

Для каждого предложения/применения хранится:

- кто подтвердил
- когда
- что применили
- исходное предложение
- результат/ошибка

## UX

- Пользователь видит список предложений:
  - предложено
  - подтверждено
  - применено
  - ошибка
  - отклонено

## Acceptance criteria

- CRM никогда не обновляется автоматически.
- Любое применённое действие имеет журнал.
- Любое предложение имеет evidence.
