# Спека 0004 — Weekly digest v1 (DRAFT)

Статус: **draft**

## Цель

Собирать еженедельную сводку по проекту, чтобы за 2–3 минуты понять:

- что произошло
- что обещано
- где риски
- что делать дальше

## Поведение

- Digest строится по периоду (неделя) автоматически и/или по кнопке.
- Digest сохраняется, чтобы можно было вернуться к прошлым версиям.
- Повторная генерация создаёт **новую версию**, а не переписывает старую.

## Структура (семантически)

- Краткое резюме (1 абзац)
- Commitments: новые / закрытые / просроченные
- Risks: новые / усиленные
- Активность переписки (объём, динамика)
- Рекомендованные действия на следующую неделю

## Evidence-first в дайджесте

- Любой пункт дайджеста должен быть:
  - либо ссылкой на commitments/risks (у которых уже есть evidence)
  - либо иметь собственные ссылки на первоисточник

Запрещено: “магические” рекомендации без основания.

## UX

- Страница digest показывает:
  - последний дайджест
  - историю
  - кнопку “сгенерировать сейчас”

## Acceptance criteria

- Digest относится к проекту и не смешивает контекст.
- Digest воспроизводим и опирается на evidence.
- История версий сохраняется.
