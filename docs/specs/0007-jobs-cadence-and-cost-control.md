# Спека 0007 — Cadence джоб и контроль стоимости (READY)

Статус: **ready**

## Факты функционала

- В системе есть повторяемые **jobs** (sync, embeddings, scheduler tick).
- Jobs можно безопасно повторять (идемпотентность).
- Система ограничивает объём работы и стоимость (лимиты, batching).
- Пользователь видит статус/ошибки выполнения.

## Инварианты

- bounded work per run (лимит на сообщения/страницы/чанки/батч)
- retry-safe (нет дубликатов)
- project-scoped выполнение

## Acceptance criteria

- Weekly cadences поддерживаются без искусственных daily caps.
- Embeddings job уважает batch size и может быть уменьшен при ошибках.
- Scheduler tick не уходит в бесконечный цикл и завершает работу с лимитом.
