# Спека 0003 — Risks v1 (DRAFT)

Статус: **draft**

## Цель

Автоматически выделять и вести **риски проекта**.

## Семантика

Risk — это прогноз/сигнал, что проект может отклониться:

- сроки
- объём
- бюджет
- настроение/удовлетворённость
- блокировки

Ключевое: риск должен быть **обоснован** (evidence-first), а не «галлюцинацией».

### Две природы риска

- **Сигнал**: красный флаг из переписки (неудовлетворённость, конфликт, scope creep).
- **Прогноз**: вероятность/влияние, но всё равно с evidence.

## Поведение

- Система извлекает риски из памяти проекта и связки с commitments.
- Каждый риск имеет:
  - “почему” (объяснение)
  - evidence
  - следующий шаг

### Ворота уверенности

- низкая/средняя уверенность → **предложенные риски**
- высокая уверенность → **активные риски**

### Жизненный цикл

- активный → смягчён
- активный → принят
- активный → закрыт (если причина исчезла)

## UX

В проекте есть список Risks с разделением:

- активные
- предложенные
- закрытые

Для каждого риска:

- тип
- серьёзность (простая шкала)
- следующий шаг
- evidence

## Наблюдаемость

- сколько рисков найдено
- сколько активировано автоматически
- сколько в предложенных
- сколько смягчено/принято

## Acceptance criteria

- Каждый активный риск содержит evidence.
- Риски строго изолированы по проектам.
- Риски без evidence не становятся “официальными”.
