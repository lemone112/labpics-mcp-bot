# Спека 0008 — Аудит, приватность, retention и безопасность (DRAFT)

Статус: **draft**

## Факты (требования)

- Система должна минимизировать риск утечек (scope enforcement).
- Должен существовать след “почему система так решила/сделала” для критичных действий (auditability).
- Должна быть политика хранения чувствительных данных (retention).

## MVP minimum

- scope enforced
- auth безопасен (см. 0017)
- runbooks описывают типовые инциденты

## Roadmap

- формальный audit log по всем действиям
- retention/cleanup jobs
- экспорт аудита для расследований
